# Strategic Performance Monitoring System Beginner Guide

## 1. What this website is for

The Strategic Performance Monitoring System is a Balanced Scorecard website that helps offices monitor KPI targets, monthly accomplishments, submission compliance, issues, and Means of Verification (MOV).

In simple terms:

- The Excel file is where accomplishment data is collected per KPI row.
- The website turns that data into dashboards, charts, tables, and summaries.
- Each dashboard answers a different management question.

## 2. Simple step-by-step guide for beginners

### Step 1: Open the website

- Go to the website login page.
- Enter your email and password.
- Click `Sign In`.

Notes:

- If the system is in demo mode, any login credentials can work.
- If you do not have an account yet, use the `Register` page first.

### Step 2: Understand the main layout

After login, you will see:

- A top header with the CSU logo and the title `Balance Scorecard`
- A left sidebar menu with all dashboards
- The main content area where the selected dashboard appears

The left sidebar contains these pages:

- Executive Dashboard
- Goal Performance
- Office Dashboard
- KPI Tracking
- Issues & Assistance
- Submission Compliance
- MOV Dashboard
- Monthly Progress
- Pillar Performance

### Step 3: Start with the Executive Dashboard

This is the best page for beginners because it gives the overall picture first.

Use it to quickly answer:

- How many KPIs are being tracked?
- How many submissions are already submitted?
- How many KPIs are completed or delayed?
- What is the overall accomplishment rate?

### Step 4: Use the other dashboards depending on your question

- If you want to see performance by goal, open `Goal Performance`.
- If you want to focus on one office only, open `Office Dashboard`.
- If you want the full KPI table and detailed target tracking, open `KPI Tracking`.
- If you want reported problems and support needs, open `Issues & Assistance`.
- If you want to monitor who submitted and who did not, open `Submission Compliance`.
- If you want to check MOV files and validation, open `MOV Dashboard`.
- If you want to review monthly progress trends, open `Monthly Progress`.
- If you want to compare performance by pillar, open `Pillar Performance`.

### Step 5: Read status colors correctly

Across the dashboards, status badges help you read the system faster:

- `Completed` means the KPI has met or finished its work
- `Ongoing` means work is still in progress
- `Delayed` means the KPI needs attention
- `Not Started` means there is still no progress
- `For Validation` means work may be done but still needs checking

For submission status:

- `Submitted` means the office submitted the required report
- `Late` means the report was submitted after the deadline
- `Not Submitted` means no report was submitted yet

## 3. Dashboard-by-dashboard explanation

## Executive Dashboard

This is the summary dashboard for leaders and first-time users.

Main contents:

- `Total KPIs`: total KPI rows loaded in the system
- `Submitted`: number of KPI rows with submitted reports
- `Completed`: number of KPI rows marked completed
- `Avg Accomplishment`: overall Q1 accomplishment percentage

Charts and sections:

- `Status Distribution`: shows how many KPIs are completed, ongoing, delayed, not started, or for validation
- `Submission Status`: shows submitted, not submitted, and late records
- `Monthly Accomplishment Trend`: shows progress by month
- `Top Performing Goals`: highest-performing goals based on target vs accomplishment
- `Offices with Missing Submissions`: offices with pending KPI submissions
- `Pillar Distribution`, `Assignment Type`, `Perspective Mix`: show how KPI rows are grouped
- `Dimension Performance Snapshot`: compares KPI count, average Q1 progress, and delayed rows by grouping
- `Data Quality Snapshot`: shows missing targets, missing MOV text, missing focal persons, no monthly updates, and other incomplete data
- `Source Traceability`: shows if KPI rows can be traced back to the source sheet and row
- `Priority KPIs`: shows KPI rows that need urgent attention

Best use:

- Start here every time you log in
- Use this page for quick reporting and management review

## 4. How the Excel file is related to the website

The workbook `Copy of CSU_Q1_Accomplishment_Collection.xlsx` has 3 sheets:

- `Instructions`
- `Q1 Accomplishment Matrix`
- `Office List`

The most important sheet is `Q1 Accomplishment Matrix`.

It has 333 KPI data rows in the template, from row 2 to row 334. Each row represents one KPI assignment for one office or unit.

This means:

- One row in Excel = one KPI record in the website
- The values entered in that row affect several dashboards at the same time

## 5. Excel sheet explanation by sheet

## Sheet 1: Instructions

This sheet explains how users should fill out the file.

Important notes from the sheet:

- Offices should filter the matrix by `Assigned Office/Unit`
- Users should update the input columns only
- `Total Q1 Accomplishment` and `% Accomplishment` are computed automatically
- Status choices are:
  `Not Started`, `Ongoing`, `Completed`, `Delayed`, `For Validation`

This sheet is directly related to the website because the website uses the same logic for status, accomplishments, and reporting fields.

## Sheet 2: Q1 Accomplishment Matrix

This is the main source sheet for the dashboards.

### Column-by-column meaning

- `Assigned Office/Unit`
  This tells which office owns the KPI row.
  Related dashboards:
  Office Dashboard, Submission Compliance, Issues, Executive Dashboard

- `Pillar`
  This groups the KPI by organizational pillar.
  Related dashboards:
  Pillar Performance, KPI Tracking filters, Executive Dashboard

- `Assignment Type`
  This shows whether the row is Strategic, Core, or Support.
  Related dashboards:
  KPI Tracking, Executive Dashboard

- `Goal`
  This connects the KPI to a strategic goal.
  Related dashboards:
  Goal Performance, Executive Dashboard

- `Perspective`
  This groups the KPI by perspective.
  Related dashboards:
  KPI Tracking, Executive Dashboard

- `Strategic Objective`
  This explains the objective behind the KPI.
  Related dashboards:
  KPI Tracking detail table

- `KPI / Strategic Measure`
  This is the KPI name shown in most tables.
  Related dashboards:
  Almost all dashboards

- `2026 Target (from BSC)`
  This is the original target from the Balanced Scorecard.
  Related dashboards:
  KPI Tracking, Goal Performance, reference context for overall review

- `Q1 Target`
  This is the quarter target used for Q1 accomplishment checking.
  Related dashboards:
  Executive Dashboard, Goal Performance, KPI Tracking, Pillar Performance

- `January`
- `February`
- `March`
  These are the monthly accomplishment inputs for Q1.
  Related dashboards:
  Monthly Progress, Office Dashboard, KPI Tracking, Executive Dashboard

- `Total Q1 Accomplishment`
  This is auto-computed from January + February + March.
  Related dashboards:
  Executive Dashboard, Goal Performance, KPI Tracking, Pillar Performance

- `% Accomplishment vs Q1 Target`
  This is auto-computed based on total accomplishment divided by Q1 target.
  Related dashboards:
  Executive Dashboard, KPI Tracking, Goal Performance, Pillar Performance

- `Key Activities / Outputs`
  This explains what was done.
  Related dashboards:
  Office Dashboard, KPI Tracking

- `Means of Verification (MOV)`
  This stores proof or evidence text/link.
  Related dashboards:
  MOV Dashboard, KPI Tracking, Executive Dashboard data quality

- `Status`
  This shows current progress status.
  Related dashboards:
  Executive Dashboard, Office Dashboard, KPI Tracking, Goal Performance

- `Issues / Challenges`
  This records problems affecting implementation.
  Related dashboards:
  Issues & Assistance, KPI Tracking

- `Assistance Needed / Recommendations`
  This records support requests or recommendations.
  Related dashboards:
  Issues & Assistance, KPI Tracking

- `Focal Person`
  This identifies the accountable person.
  Related dashboards:
  Office Dashboard, Submission Compliance, Executive Dashboard data quality

- `Submission Date`
  This records when the office submitted.
  Related dashboards:
  Submission Compliance, Office Dashboard, KPI Tracking

- `BSC Remarks`
  This stores special notes about the KPI row from the original BSC.
  Related dashboards:
  KPI Tracking

- `Source Sheet`
  This shows where the row came from in the source workbook.
  Related dashboards:
  Executive Dashboard source traceability, KPI Tracking

- `Source Row`
  This shows the original row number.
  Related dashboards:
  Executive Dashboard source traceability, KPI Tracking

### Row meaning in the matrix

Each row is one reporting line for one office or unit.

Example of how one row behaves:

- The `Goal` value sends it to the Goal Performance Dashboard
- The `Assigned Office/Unit` value sends it to the Office Dashboard and Compliance Dashboard
- The `Pillar` value sends it to the Pillar Performance Dashboard
- The `January`, `February`, and `March` values affect monthly charts and Q1 totals
- The `Status` affects executive summary counts
- The `Issues / Challenges` and `Assistance Needed / Recommendations` affect problem monitoring
- The `MOV` affects evidence completeness tracking

So one Excel row can appear in many dashboards at once.

### Important workbook behavior

In your Excel template:

- `Total Q1 Accomplishment` uses a formula like `SUM(J:L)`
- `% Accomplishment vs Q1 Target` uses a formula like `IFERROR(M/I,"")`

This means users should usually type values into:

- `Q1 Target`
- `January`
- `February`
- `March`
- `Key Activities / Outputs`
- `Means of Verification (MOV)`
- `Status`
- `Issues / Challenges`
- `Assistance Needed / Recommendations`
- `Focal Person`
- `Submission Date`

They should not manually compute the total and percentage columns.

## 6. How data flows from Excel to the website

Simple flow:

1. Offices update the `Q1 Accomplishment Matrix`
2. The system reads the matrix data
3. Each KPI row becomes a website KPI record
4. Monthly values become monthly accomplishment records
5. MOV entries become evidence-related records
6. Issues and assistance notes become monitoring content
7. Dashboards summarize the records into charts and tables

In the code, the website is designed to work with these core data groups:

- Goals
- Offices
- KPIs
- Monthly Accomplishments
- Issues
- MOVs

That is why one KPI row can influence several pages.

## 7. Easy way to explain the dashboards to beginners

If you are teaching new users, use this order:

1. Login
2. Executive Dashboard for the big picture
3. Office Dashboard for their own office
4. Submission Compliance to check reporting
5. KPI Tracking for detailed checking
6. MOV Dashboard for evidence
7. Issues Dashboard for problems needing action
8. Monthly and Pillar dashboards for analysis
9. Goal Dashboard for strategic alignment

This order is easier because users move from simple summary to detailed analysis.

## 8. Tips for using your website

- Start with `Executive Dashboard` before opening detailed pages.
- Use `Office Dashboard` when talking to one office only.
- Use `Submission Compliance` every reporting period to follow up missing submissions.
- Use `KPI Tracking` when validating figures, targets, and remarks.
- Use `MOV Dashboard` to check if evidence is complete before final review.
- Use `Issues & Assistance` to identify which problems need leadership support.
- Always fill in `Focal Person` and `Submission Date` to make follow-up easier.
- Avoid leaving `Q1 Target` blank because it affects percentage computation.
- Enter monthly values consistently because January, February, and March directly affect Q1 reports.
- Keep status updated so the executive summary remains accurate.
- Use clear MOV text or links so validators can confirm evidence quickly.
- Review rows with missing data because they reduce report quality.

## 9. Beginner reminders

- One row in Excel is very important because it feeds many dashboard views.
- Blank fields can cause incomplete charts, weak compliance tracking, or misleading totals.
- If a row has the wrong office, goal, or pillar, it will appear in the wrong dashboard group.
- If monthly accomplishments are blank, the KPI may look like it has no progress.
- If MOV is blank, the KPI may appear incomplete even if work was done.

Your website is structured well for reporting because it separates monitoring into clear dashboard views:

- Executive view for management
- Goal and pillar views for strategy
- Office and KPI views for operations
- Compliance, issues, and MOV views for follow-up and validation

Your Excel workbook is also aligned with the website because the matrix contains the same core reporting fields the dashboards need:

- ownership
- strategic grouping
- target values
- monthly accomplishment values
- status
- issues
- assistance requests
- evidence
- submission details

The strongest relationship in your system is this:

- the `Q1 Accomplishment Matrix` is the operational input
- the website dashboards are the reporting output

Because of that, the quality of the website depends heavily on complete and correct Excel row entries.
